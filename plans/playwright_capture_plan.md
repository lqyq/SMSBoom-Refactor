# Playwright 抓包录入方案 - 架构设计文档

## 1. 方案概述

**目标：** 使用 Playwright 在真实浏览器中打开搜索引擎或目标网站，用户手动浏览并操作发送验证码页面，后台自动抓取网络请求并智能识别发送短信的 API，最终将其转换为项目标准格式录入数据库。

**结论：✅ 方案完全可行**

### 交互流程升级：搜索 -> 浏览 -> 抓包

用户不再需要逐个输入 URL，而是可以在浏览器中利用搜索引擎批量发现目标，系统持续监听并捕获发码请求。

flowchart TD

    L --> M[自动替换手机号为 phone 占位符]

## 4. 核心模块设计

### 4.1 `playwright_capture.py` - 抓包引擎

```python
class PlaywrightCapture:
    """Playwright 网络请求捕获器"""
    
    属性:
        browser         # 浏览器实例
        context         # 浏览器上下文 (关键：监听此级别的事件)
        phone           # 用户输入的测试手机号（用于过滤）
        event_queue     # 线程安全的事件队列
    
    方法:
        start(phone, initial_url=None)  # 启动浏览器，可选打开初始页
        stop()                          # 停止监听并关闭
        _setup_context_listeners()      # 监听 context 的 page 事件
        _on_page(page)                  # 新页面监听回调
        _on_request(req)                # 请求监听回调
```

#### 核心逻辑 - 上下文监听：

```python
# 监听新页面创建（用户点击链接打开新标签页时）
context.on("page", lambda page: page.on("request", self._on_request))

# 初始页面也要监听
page = context.new_page()
page.on("request", self._on_request)
```

#### 核心逻辑 - 请求过滤策略（不变）：

1. **必要条件**：请求 URL 或 Body 中包含用户输入的手机号
2. **排除条件**：静态资源、分析跟踪、已知第三方
3. **优先标记**：URL 关键词
